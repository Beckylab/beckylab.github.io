1 全流程概览

自动投标 Agent 覆盖从标讯发现到标书提交的完整链路，各环节由独立 Agent 负责，通过编排层串联：

标讯采集
抓取各平台招标公告
文件解析
提取资质、评分标准
投标决策
Go / No-Go 评估
标书生成
技术标 + 商务标
合规审查
逐项核验防废标
提交封装
按平台要求打包上传
2 架构建议

推荐多 Agent 协作架构：一个编排 Agent 统一调度，各专业 Agent 各司其职，便于独立迭代、故障隔离和权限管控。

2.2 各 Agent 职责
编排

项目经理 Agent

流程"大脑"。将招标文件拆解为子任务，按依赖关系分发，跟踪状态，处理异常重试，支持可配置流程模板。

解析

文档解析 Agent

将 PDF/Word/扫描件转为结构化 JSON：项目信息、资质要求、评分标准、技术参数、格式要求。

决策

决策评估 Agent

Go/No-Go 评分卡：资质匹配度、历史中标率、预估利润率、竞争对手分析、资源可用性。

核心

标书撰写 Agent

从知识库 RAG 检索素材，按模板逐章节生成技术标和商务标，逐条响应评分标准，保持企业风格一致。

审查

合规审查 Agent

将硬性要求拆解为 checklist 逐项核验，关键数据交叉校验，输出审查报告标记风险项。宁可误报不可漏报。

策略

定价策略 Agent

基于历史中标数据和成本结构，分析同类项目价格区间，给出推荐报价范围。辅助性质，最终人工定价。

2.3 数据流转

各 Agent 之间通过标准化的数据协议进行通信。一次完整的投标流程，数据流转如下：

Step 1 招标文件 → 文档解析 Agent → 输出结构化需求 JSON（资质清单、评分标准、技术参数、格式要求）
Step 2 结构化需求 → 决策评估 Agent → 输出 Go/No-Go 评分卡（人工确认是否继续）
Step 3 需求 + 企业知识库 → 标书撰写 Agent → 通过 RAG 检索素材，逐章节生成标书初稿
Step 4 标书初稿 + 原始需求 → 合规审查 Agent → 输出合规审查报告（通过/风险项清单）
Step 5 审查通过 → 定价策略 Agent → 输出报价建议 → 人工终审 → 封装提交

关键设计原则：每个 Agent 的输入输出都定义为标准化 Schema，通过 JSON Schema 校验，确保上下游数据格式一致。编排 Agent 维护全局状态机，支持断点续传和人工介入。

2.5 安全设计
数据

隔离与加密

多租户逻辑/物理隔离，TLS 传输 + AES-256 存储加密，定期备份。

权限

RBAC + 审批

角色分级管控，关键操作多人审批，完整审计日志。

模型

LLM 调用安全

敏感数据走私有化模型，云端模型确保不出境不训练，Prompt 注入防护。

部署

私有化交付

Docker Compose / K8s，支持内网离线运行，信创环境适配。

4 落地路径
第一阶段 · 1-2 月

MVP — 文件解析 + 标书生成

招标文件自动解析 + 标书初稿生成，半自动模式，聚焦核心价值验证。

第二阶段 · 2-3 月

增强 — 知识库 + 合规审查

RAG 知识库上线，合规自动审查，模板管理支持多行业。

第三阶段 · 3-6 月

扩展 — 全流程 + 智能化

标讯采集接入，决策评估 + 定价策略上线，多用户协作与审批。
